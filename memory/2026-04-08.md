## WEB 端重设计计划

### 问题分析
- ❌ 移动端优先 UI 在 WEB 端显示元素过大
- ❌ 底部导航不适合桌面端
- ❌ 单列布局浪费桌面空间
- ❌ 缺少悬停效果、快捷键等桌面交互

### 设计方案
**推荐方案**: 顶部导航栏 + 响应式布局

**核心调整**:
- 导航：底部导航 → 顶部导航栏 (64px)
- 布局：单列 → 3-4 列网格 (桌面端)
- 容器：max-w-4xl → max-w-7xl (1280px)
- 字体：text-2xl → text-3xl (桌面端)
- 按钮：py-4 (44px) → py-2.5 (40px)
- 圆角：rounded-2xl (16px) → rounded-xl (12px)

**响应式断点**:
- sm: 640px (手机横屏)
- md: 768px (平板)
- lg: 1024px (小笔记本)
- xl: 1280px (桌面)
- 2xl: 1536px (大桌面)

### 实施计划
- **P0**: 顶部导航栏 + 响应式断点配置
- **P1**: 首页/模板中心网格布局优化
- **P2**: 表单页面适配
- **P3**: 快捷键 + 交互增强
- **预计工期**: 5-8 天
