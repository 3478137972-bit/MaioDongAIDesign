# 图片查看器 UI 交互设计 - 执行摘要

## 📌 项目概述

本文档是图片查看器 UI 交互设计方案的执行摘要，为开发团队提供快速参考。

---

## 🎯 核心功能模块

### 1️⃣ 画廊模式布局

**设计要点:**
- 响应式网格布局 (3-8 列，根据设备调整)
- 图片比例自适应 (1:1 / 4:3 / 16:9)
- 无限滚动加载 (距底部 300px 触发)
- 懒加载优化 (Intersection Observer)

**关键参数:**
```
手机：3 列，间距 4px，边距 8px
平板：4-6 列，间距 8px，边距 16px
桌面：6-8 列，间距 12px，边距 24px
```

---

### 2️⃣ 缩放交互动画

**设计要点:**
- 双击缩放：100% ↔ 200% (以点击点为中心)
- 捏合缩放：50% - 400% 无级调节
- 弹性边界：超出 15% 后回弹
- 动画曲线：cubic-bezier(0.34, 1.56, 0.64, 1)

**动画配置:**
```javascript
{
  duration: 300ms,
  easing: 'cubic-bezier(0.34, 1.56, 0.64, 1)',
  boundaryOvershoot: 0.15,
  bounceDuration: 400ms
}
```

---

### 3️⃣ 手势操作设计

**手势映射表:**

| 手势 | 操作 | 优先级 |
|------|------|--------|
| 双指捏合 | 无级缩放 | ⭐⭐⭐ (最高) |
| 双指拖拽 | 旋转 (可选) | ⭐⭐ |
| 单指拖拽 | 平移图片 | ⭐⭐ |
| 水平滑动 | 切换图片 | ⭐ |
| 双击 | 快速缩放 | ⭐ |
| 单击 | 切换 UI | ⭐ |
| 长按 | 操作菜单 | ⭐ |

**手势检测参数:**
```javascript
{
  doubleTap: { maxInterval: 300ms, maxDistance: 20px },
  swipe: { minDistance: 50px, maxAngle: 30° },
  longPress: { duration: 500ms, maxMove: 10px }
}
```

---

### 4️⃣ 响应式适配方案

**断点设计:**
```css
--breakpoint-sm: 576px   /* 小手机 */
--breakpoint-md: 768px   /* 大手机/平板 */
--breakpoint-lg: 992px   /* 小桌面 */
--breakpoint-xl: 1200px  /* 大桌面 */
--breakpoint-xxl: 1600px /* 超大屏 */
```

**适配策略:**
- 移动优先 (Mobile First)
- 横竖屏自动切换
- iOS 安全区域适配 (safe-area-inset)
- 图片资源按需加载 (根据视口尺寸)

## 🔄 核心交互流程

```
用户打开相册
    ↓
加载画廊网格 (首屏 + 懒加载)
    ↓
┌───────────────────────┐
│     用户操作分支       │
├───────────────────────┤
│ 单击图片 → 进入预览    │
│ 双击图片 → 进入预览    │
│ 长按图片 → 多选模式    │
│ 滑动页面 → 加载更多    │
└───────────────────────┘
    ↓
预览模式
    ↓
┌───────────────────────┐
│     预览操作分支       │
├───────────────────────┤
│ 双击 → 放大/还原       │
│ 捏合 → 无级缩放        │
│ 拖拽 → 平移 (放大时)   │
│ 左/右滑 → 切换图片     │
│ 单击 → 切换控制栏      │
│ 长按 → 操作菜单        │
│ 关闭 → 返回画廊        │
└───────────────────────┘
```

## 🛠️ 技术实现建议

### 推荐技术栈

```
框架：React / Vue 3 / Svelte
状态管理：Zustand / Pinia
手势库：@use-gesture/react / @vueuse/gesture
动画库：Framer Motion / GSAP
图片处理：react-image-gallery / 自定义 Canvas
性能优化：Intersection Observer + Web Workers
```

### 关键代码示例

```jsx
// 手势处理 (React + @use-gesture)
const bind = useGesture({
  onPinch: ({ da: [d] }) => api.start({ scale: clamp(1 + d, 0.5, 4) }),
  onDrag: ({ movement: [x, y] }) => scale > 1 && api.start({ x, y }),
  onDoubleClick: () => api.start({ scale: scale === 1 ? 2 : 1, x: 0, y: 0 })
})
```

## 🎨 设计原则

1. **流畅性**: 所有动画 60fps，手势响应 < 100ms
2. **一致性**: 跨平台交互逻辑统一
3. **可访问性**: 支持键盘导航和屏幕阅读器
4. **性能优先**: 懒加载、虚拟滚动、资源优化
5. **渐进增强**: 基础功能全平台可用，高级功能按需启用

## 💡 注意事项

### 开发注意
- 手势冲突按优先级解决 (双指 > 单指，缩放 > 平移)
- 图片加载使用渐进式 (缩略图 → 预览图 → 原图)
- 内存管理：最大缓存 10 张图片，超出自动卸载

### 设计注意
- 避免过度动画，保持简洁
- 边界反馈要明显但不过分
- 工具栏自动隐藏逻辑要清晰

### 测试注意
- 真机测试必不可少 (尤其是手势)
- 低端设备性能测试
- 网络慢速场景测试
